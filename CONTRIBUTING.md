# Contributing to SingleFileTests

Thank you for your interest in contributing to SingleFileTests!

## How to Contribute

### Reporting Issues

- Check if the issue already exists before creating a new one
- Include .NET version and OS information
- Provide steps to reproduce the issue

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-new-feature`)
3. Make your changes
4. Test your changes by running the samples with `dotnet run <filename.cs>`
5. Commit your changes (`git commit -am 'Add new feature'`)
6. Push to the branch (`git push origin feature/my-new-feature`)
7. Create a Pull Request

### Adding New Samples

When adding new sample files to the `/samples` directory:

- Follow the pattern established in `_template.cs`
- Include comments explaining what the sample demonstrates
- Update the README.md with a description of your sample
- Ensure the sample runs successfully with `dotnet run`

## Code of Conduct

Please be respectful and constructive in all interactions.

## Questions?

Feel free to open an issue if you have questions about contributing.
